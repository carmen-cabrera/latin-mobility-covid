# Figure reproducibility guide

This document maps manuscript figures and supplementary figures to the notebooks that generate the underlying analysis outputs or source plot panels.

The Meta-Facebook mobility and population datasets used in the study cannot be redistributed. For that reason, the public repository cannot regenerate figures end-to-end without approved access to the restricted data. In addition, several final manuscript figures are composite figures assembled manually from locally generated panels. The purpose of this guide is to make that workflow explicit: it identifies the scripts used to produce the source data, plot panels, model outputs, or tables that were then combined into the final manuscript figures.

Unless otherwise noted, generated outputs are written locally under `data/outputs/` and `plots/`. These output folders are not intended to be complete in the public repository because they may depend on restricted input data or local figure assembly.

## Notes On Manual Figure Assembly

Several final manuscript figures were made by exporting source panels locally and arranging them into multi-panel figures outside the codebase. This affects figure layout, cropping, labels and final visual composition, but not the underlying statistical outputs. The underlying data processing, model fitting, trend extraction, recovery-time estimation and raw-data sensitivity checks are implemented in the notebooks listed above.

For reviewer-facing reproducibility, the relevant distinction is:

- Reproducible with restricted data access: preprocessing outputs, trend extraction, model outputs, recovery-time estimates, raw-versus-imputed sensitivity summaries and most individual source panels.
- Documented but not code-generated end-to-end: final composite manuscript figures, manually formatted table images and conceptual workflow diagrams.
